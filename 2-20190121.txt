 * Method Overriding
   : 슈퍼 클래스와 서브 클래스의 메소드에서 발생
   : 슈퍼 클래스의 메소드를 서브 클래스에서 재정의하는 것
   : 슈퍼 클래스의 메소드 이름, 메소드 인자 타입 및 개수, 리턴 타입 등 모든 것 동일하게 정의
   : static, private, 또는 final 메소드는 오버라이딩 될 수 없다
   : 오버라이딩된 메소드의 접근 지정자는 슈퍼 클래스의 메소드의 접근 지정자 보다 좁아질 수 없다


 * 오버로딩
   - 같은클래스
   - 같은 이름의 메소드가 여러개 정의
   - 메소드의 매개변수의 타입과 갯수는 달라야 한다

 * 오버라이딩
   - 상속관계의 클래스에서 부모의 메소드가 자식에 재정의 하는것
   - 메소드원형(리턴형,메소드이름,매개변수)이 같아야 함


 * 업캐스팅
   - Parent p = new Children();
   - 타입을 부모타입으로 정의하고 객체는 자식객체로 생성
   - 오버라이딩된 메소드가 정의되어있는 경우 동적바인딩을 통해 자식메소드를 호출한다
   - 자식메소드는 오버라이딩된 메소드만 호출할 수 있다


 * 객체 형변환       -----------★
   : 상속 관계에서는 부모자식간에 형변환이 가능
      class Person {}
      class Student extends Person {}
      Student s = new Student(); 
      Person p = s; // 업캐스팅, 자동타입변환
      Person p2 = new Student();

 * 업캐스팅
   : 서브 클래스 객체가 슈퍼 클래스 타입으로 변환

 * 다운 캐스팅
   : 업캐스팅된 것을 다시 원래대로 되돌리는 것을 다운 캐스팅

 * 레퍼런스가 어떤 객체의 타입인지 구별하기 위해서 instanceof 키워드를 이용
